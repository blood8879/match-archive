# Product Requirements Document (PRD)

> [!IMPORTANT]
> **MVP Capsule**
> - **Product**: Match Archive (아마추어 축구 기록/매칭 관리)
> - **Core Value**: 흩어진 아마추어 축구 기록을 체계적으로 관리하고, 데이터 기반으로 즐기는 축구 라이프
> - **Target Persona**: 아마추어 축구팀 운영진(주장/총무) 및 열정적인 팀원
> - **MVP Goal**: 팀 생성, 멤버 관리, 친선 경기 기록 및 개인 스탯 자동화 (리그 리모델링 제외)
> - **Key Features**: 팀 관리, 경기 생성/기록 입력, 개인/팀 스탯 대시보드
> - **North Star Metric**: Weekly Active Teams (주간 기록 생성 팀 수)
> - **Input Metrics**: 신규 가입 팀 수, 경기당 기록 완료율
> - **Non-goals**: 리그 생성/운영 기능, 결제/유료화, 실시간 중계, 매칭 용병 구인 게시판(단순 기록용 용병만 허용)
> - **NFR**: 데이터 무결성(스코어=기록합), 모바일 반응형 최적화
> - **Data Sensitivity**: 이름, 프로필 사진 (공개 원칙)
> - **Top Risk**: 기록 입력의 귀찮음으로 인한 이탈 -> 간편 입력 UX 필수
> - **Next Action**: 핵심 UX (기록 입력) 프로토타이핑

## 1. Problem & Goals
### Problem
- 아마추어 축구 기록이 카카오톡, 밴드, 엑셀 등에 흩어져 있어 관리가 어렵다.
- "내가 몇 골 넣었지?", "우리 팀 승률이 얼마지?"를 알기 어렵다.
- 용병으로 뛰거나 팀을 옮길 때 기존 기록이 유실된다.

### Goals
- **Record**: 쉽고 정확한 경기 기록 저장소 제공
- **Archive**: 개인의 축구 인생(Career)을 증명할 수 있는 데이터 아카이브
- **Manage**: 팀 운영진의 회원/출석/회비(추후) 관리 부담 경감

## 2. User Stories (MVP Scope)

### EPIC-1: 팀 & 멤버 (Team & Member)
- **FEAT-1.1 (팀 생성)**: 사용자는 팀명, 엠블럼, 활동 지역을 입력하여 팀을 생성할 수 있다.
- **FEAT-1.2 (팀 가입/초대)**:
  - 사용자는 팀을 검색하여 가입 신청을 할 수 있다. (공개 가입)
  - 운영진은 초대 링크를 생성하여 팀원을 초대할 수 있다. (비공개 가입)
  - 운영진은 가입 신청을 승인/거절할 수 있다.
- **FEAT-1.3 (용병 관리)**: 운영진은 비회원(용병)을 팀 멤버 리스트에 '게스트' 타입으로 임시 등록할 수 있다. 추후 해당 용병이 가입 시 기록 병합(Merge)을 지원한다.

### EPIC-2: 경기 & 기록 (Match & Stats)
- **FEAT-2.1 (경기 생성)**: 운영진은 일시, 장소, 상대팀(입력 없으면 'Unknown') 정보를 입력하여 경기를 생성한다.
- **FEAT-2.2 (라인업 및 쿼터)**: 운영진은 쿼터별 출전 선수(선발/교체)를 설정한다. (MVP: 단순히 뛴 선수 체크만 가능하게 단순화 권장)
- **FEAT-2.3 (결과 입력)**:
  - 홈팀(경기 생성자)이 스코어와 주요 기록(득점, 어시스트)을 입력한다.
  - 상대팀 승인 절차 없이 즉시 반영된다 (친선 경기).
  - **Validation**: (A팀 득점 + B팀 자책골) == A팀 최종 점수 검증 로직 적용.
- **FEAT-2.4 (자책골 처리)**: 자책골 발생 시, '득점자: 기록 없음(또는 상대팀)', '발생 시점', '팀 스코어: 상대팀+1' 로 처리한다.

### EPIC-3: 대시보드 (Dashboard)
- **FEAT-3.1 (팀 라커룸)**: 팀의 최근 5경기 결과(W/D/L), 통산 승률, 다음 경기 일정을 보여준다.
- **FEAT-3.2 (마이 페이지)**:
  - 나의 통산 골, 어시스트, 경기 수, MOM 횟수.
  - "클린 시트" (골키퍼 포지션인 경우 노출).

## 3. Assumptions & Dependencies
- **Assumption**: 운영진은 경기 직후 24시간 이내에 기록을 입력할 동기가 충분하다 (팀원들의 닥달 등).
- **Dependency**: Supabase의 무료 플랜(Storage, DB) 한도 내에서 MVP 운영 가능.

## 4. Open Questions / Risks
- **Q**: 같은 날 여러 경기를 뛸 때(더블 헤더) UI 처리는? -> A: 시간순 정렬로 리스트 노출.
- **Risk**: 상대 팀이 우리 팀 기록을 허위라고 주장할 경우 분쟁 해결 방안? -> Mitigation: MVP에서는 분쟁 해결 기능 없음 (홈팀 입력 데이터 신뢰). 추후 '이의 제기' 기능 도입 고려.
